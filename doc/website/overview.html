What is Grappa?

Many emerging problems are irregular, with little locality and imbalanced work, leading to inefficient execution at scale. Social networks are one example.

Performance on these problems is artificially limited by processors' dependence on locality for performance. There are two main challenges:

  • Latency in chasing down low-locality data
  • Load imbalance due to irregularity

Grappa is a parallel runtime system that leverages the parallelism inherent in these applications to get faster performance on mass-market clusters.

Key idea: Trade latency for throughput

Grappa’s core component is a lightweight cooperative threading system. We tolerate latency by context-switching to other work. Given this ability, we can increase throughput by increasing the latency of key operations:

  • Context-switch on long-latency memory operations
  • Delay and aggregate messages for better utilization of network bandwidth
  • Migrate synchronization operations to avoid unnecessary serialization
